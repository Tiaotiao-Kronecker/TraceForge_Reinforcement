# 可见性掩码（Visibility Mask）作用机制详解

## 一、用户理解验证

**用户理解**：400个查询点，可能最终跟踪的个数是不足400个的？

**答案**：**不完全正确**。所有400个点的轨迹都会保存，但每个点在每个时间步的可见性由掩码标记。

---

## 二、可见性掩码的生成过程

### 2.1 模型输出

**位置**：`utils/inference_utils.py` 第173-175行

```python
coords, visib_logits = preds.coords, preds.visibs
# coords形状: (T, 400, 3) - 每个时间步每个点的3D坐标
# visib_logits形状: (T, 400) - 每个时间步每个点的可见性logits（原始输出）

visibs = torch.sigmoid(visib_logits) >= vis_threshold
# visibs形状: (T, 400) - 每个时间步每个点的可见性（布尔值）
# vis_threshold = 0.9（默认值）
```

**关键步骤**：
1. **模型输出**：`visib_logits`（原始logits值，范围：-∞到+∞）
2. **Sigmoid激活**：`sigmoid(visib_logits)`（转换为0-1之间的概率）
3. **阈值判断**：`>= 0.9`（如果概率≥0.9，则认为可见）

**示例**：
```python
visib_logits = [-2.0, 0.5, 3.0, 1.5]  # 模型原始输出
sigmoid = [0.12, 0.62, 0.95, 0.82]     # 转换为概率
visibs = [False, False, True, False]   # 阈值0.9判断
```

---

### 2.2 可见性判断标准

**可见性阈值**：`vis_threshold = 0.9`（默认值）

**判断逻辑**：
- `sigmoid(visib_logits) >= 0.9` → **可见**（True）
- `sigmoid(visib_logits) < 0.9` → **不可见**（False）

**为什么使用0.9这个高阈值？**
- 确保只有**高置信度**的点被标记为可见
- 减少误判（将不可见的点误判为可见）
- 提高跟踪质量

---

## 三、可见性掩码的作用

### 3.1 标记点的可见性

**数据结构**：
```python
visibs: (T, 400)  # 布尔数组
# visibs[t, n] = True  表示第n个点在第t个时间步可见
# visibs[t, n] = False 表示第n个点在第t个时间步不可见
```

**示例**：
```python
visibs = [
    [True, True, False, True, ...],   # 时间步0：点0,1,3可见，点2不可见
    [True, False, False, True, ...],  # 时间步1：点0,3可见，点1,2不可见
    [False, False, False, True, ...], # 时间步2：只有点3可见
    ...
]
```

### 3.2 不可见的原因

**点不可见的情况**：
1. **超出图像范围**：点移动到图像外
2. **被遮挡**：点被其他物体遮挡
3. **跟踪失败**：模型无法找到对应的特征
4. **深度无效**：该位置没有有效的深度值
5. **置信度低**：模型预测的可见性概率 < 0.9

### 3.3 坐标与可见性的关系

**重要**：即使点不可见，**坐标仍然存在**：

```python
coords: (T, 400, 3)  # 所有400个点的坐标都存在
visibs: (T, 400)     # 标记哪些坐标是有效的

# 不可见的点，坐标可能是：
# - 无效值（如-inf, nan）
# - 超出图像范围的坐标
# - 跟踪失败时的最后已知位置
```

## 五、可见性掩码的具体作用机制

### 5.1 在模型内部

**模型预测可见性**（基于Transformer架构）：

```python
# 在模型内部（BaseTrackerPredictor）
# 1. 提取特征
query_track_feat = sample_features4d(fmaps[:, 0], coords[:, 0])

# 2. 计算相关性
fcorrs = fcorr_fn.corr_sample(track_feats, coords)

# 3. Transformer处理
transformer_input = torch.cat([flows_emb, fcorrs_, track_feats_], dim=2)
x = transformer(transformer_input)

# 4. 预测可见性logits
visib_logits = model.predict_visibility(x)  # 模型输出
```

**可见性预测依据**：
1. **特征匹配**：查询点特征与当前帧特征的相似性
2. **相关性强度**：相关性值越高，可见性越高
3. **位置合理性**：坐标是否在图像范围内
4. **深度有效性**：该位置是否有有效的深度值

### 5.2 在推理过程中

**迭代优化**（`num_iters=6`次迭代）：

```python
for iteration in range(num_iters):
    # 1. 预测坐标更新
    delta_coords = model.predict_coords(...)
    coords = coords + delta_coords
    
    # 2. 预测可见性更新
    delta_visibs = model.predict_visibility(...)
    visibs = visibs + delta_visibs  # 累积更新
    
    # 3. 根据可见性调整坐标
    # 不可见的点可能坐标无效或超出范围
```

**每次迭代**：
- 可见性预测会**逐步优化**
- 不可见的点可能被**标记为无效**
- 可见的点会**继续跟踪**

### 5.3 在最终输出中

**返回结果**（`utils/inference_utils.py` 第174-175行）：

```python
visibs = torch.sigmoid(visib_logits) >= vis_threshold
return coords.squeeze(), visibs.squeeze()
# coords: (T, 400, 3) - 所有400个点的坐标
# visibs: (T, 400) - 所有400个点的可见性
```

**关键点**：
- **所有400个点都有坐标和可见性信息**
- 不可见的点，坐标可能无效（但数据结构保持400个）

## 六、实际数据验证

### 6.1 从NPZ文件分析

**实际保存的数据**：
```python
{
    "traj": (400, 128, 3),      # 所有400个点的轨迹
    "keypoints": (400, 2),      # 所有400个点的起始关键点
    "valid_steps": (128,),      # 轨迹重定向后的有效时间步
}
```

**关键发现**：
- **轨迹数量始终是400个**
- 每个轨迹有128个时间步（重定向后）
- 某些时间步可能无效（标记在`valid_steps`中）

### 6.2 无效值的处理

**轨迹重定向后**（`retarget_trajectories`函数）：

```python
retargeted, valid_mask = retarget_trajectories(traj, max_length=128)
# retargeted: (400, 128, 3) - 所有400个点，每个128个时间步
# valid_mask: (128,) - 标记哪些时间步有效

# 无效的时间步用 -np.inf 填充
retargeted[:, invalid_steps, :] = -np.inf
```

**结果**：
- 所有400个点的轨迹都保存
- 无效的时间步用`-np.inf`填充
- `valid_steps`标记哪些时间步有效

## 七、总结

### 7.1 可见性掩码的作用

1. **标记可见性**：每个点在每个时间步是否可见
2. **质量控制**：只有高置信度的点被标记为可见（阈值0.9）
3. **跟踪状态**：反映点的跟踪状态（成功/失败）

### 7.2 最终保存的点数

**答案**：**所有400个点的轨迹都会保存**

**原因**：
1. 模型总是返回400个点的轨迹
2. 保存逻辑不区分可见性，所有点都保存
3. 不可见的点用无效值（`-np.inf`）填充

### 7.3 可见性信息的使用

**保存的数据**：
- `traj`: (400, T, 3) - 所有400个点的轨迹
- `valid_steps`: (T,) - 轨迹重定向后的有效时间步掩码
- **不直接保存**`visibs`，但可以通过无效值推断

**使用方式**：
- 检查`traj`中的无效值（`-np.inf`, `nan`）来识别不可见的点
- 使用`valid_steps`来标记哪些时间步有效
- 在可视化时，可以根据可见性过滤点

### 7.4 用户理解的纠正

**❌ 错误理解**：最终跟踪的点数可能少于400个

**✅ 正确理解**：
- **所有400个点的轨迹都会保存**
- 但每个点在每个时间步的可见性不同
- 不可见的点用无效值填充，但数据结构保持400个
- 在可视化或使用时，可以根据可见性过滤点

## 九、实际应用建议

### 9.1 如何过滤不可见的点？

**方法1**：检查无效值
```python
# 加载轨迹
traj = data["traj"]  # (400, 128, 3)

# 检查无效值
valid_mask = ~np.isinf(traj).any(axis=2)  # (400, 128)
valid_points = valid_mask.any(axis=1)     # (400,) - 哪些点至少有一个有效时间步

# 只使用有效的点
valid_traj = traj[valid_points]  # (N_valid, 128, 3)
```

**方法2**：使用valid_steps
```python
valid_steps = data["valid_steps"]  # (128,)
# 只使用有效时间步的轨迹
valid_traj = traj[:, valid_steps, :]
```

### 9.2 统计可见点数

```python
# 统计每个时间步的可见点数
visible_counts = valid_mask.sum(axis=0)  # (128,)

# 统计每个点的可见时间步数
point_visible_counts = valid_mask.sum(axis=1)  # (400,)

# 找出完全不可见的点
invisible_points = point_visible_counts == 0  # (400,)
```

## 十、关键结论

1. **可见性掩码**：标记每个点在每个时间步是否可见（阈值0.9）
2. **保存点数**：**所有400个点的轨迹都会保存**，不减少
3. **无效值处理**：不可见的点用`-np.inf`填充，但数据结构保持400个
4. **使用建议**：在可视化或使用时，可以根据可见性过滤点

**核心答案**：
- **最终保存的点数始终是400个**
- 但每个点在每个时间步的可见性不同
- 不可见的点用无效值填充，但不会从数据结构中移除
