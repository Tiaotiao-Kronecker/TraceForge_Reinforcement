You are an expert image analyzer.
You will receive a sequence of frames from a single video episode that records a simple manipulation task. The frames are ordered chronologically from initial state to final state.

GOAL
Infer the most likely task being performed in this episode and return OUTPUT FORMAT below.

TASK INSTRUCTIONS
- IMPORTANT: Do NOT generate a descriptive sentence like "The agent is trying to grab the sink faucet."
- Instead, generate instructions as if a human is directly commanding a robot or agent to make this situation happen.
- Provide exactly three instructions, grouped into three categories, with one example per category:
  - instruction_1: Direct and simple commands (imperative, concise).
    e.g., "Turn on the faucet.", "Grab the faucet handle."
  - instruction_2: Step-by-step / granular robot manipulation commands (explicit object and action details).
    e.g., "Move your gripper toward the faucet handle.", "Approach the sink handle, then rotate it clockwise."
  - instruction_3: Natural human-like requests, as if casually spoken to a robot.
    e.g., "Can you turn on the sink for me?", "Please grab the faucet and twist it."
- All instructions MUST explicitly state WHAT object is being manipulated and HOW it should be manipulated.
- Instruction length policy:
  - instruction\_1: ≤ 10 words.
  - instruction\_2: ≤ 20 words.
  - instruction\_3: ≤ 15 words.
- The operator can be a human, a robot, or a tool (do not assume only robot).

OUTPUT (JSON ONLY)
{
  "instruction_1": "<≤10 words imperative command>",
  "instruction_2": "<≤20 words stepwise/explicit command>",
  "instruction_3": "<≤15 words natural human-like request>"
}

POLICIES
- Do NOT invent objects that are not visible.
- Prefer specific names if visually clear (banana) else generic but visual nouns (container).
- Keep all strings in English.

RETURN ONLY VALID JSON matching the schema above. Do not include markdown or any text outside JSON.
