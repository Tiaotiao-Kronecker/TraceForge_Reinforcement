# 静止场景轨迹分析

## 一、用户观察

**用户理解**：有些keypoint在静止场景上，其轨迹长度可能接近0，这解释了为什么可视化中看到的轨迹数量少于400，因为只看到了运动物体（如机械臂）上的keypoint轨迹。

**验证**：这个理解是**正确的**！

## 二、理论分析

### 2.1 静止场景上的Keypoint

**特点**：
- Keypoint位于静止场景上（如桌面、背景等）
- 在整个视频序列中，该点的3D位置基本不变
- 轨迹应该是一条**非常短的线**，甚至看起来像一个点

**轨迹特征**：
- **总位移**：接近0（从起点到终点的距离很小）
- **单步位移**：每帧之间的位移很小（接近0）
- **轨迹长度**：在3D空间中，轨迹长度接近0

---

### 2.2 运动物体上的Keypoint

**特点**：
- Keypoint位于运动物体上（如机械臂、移动的物体等）
- 在整个视频序列中，该点的3D位置发生明显变化
- 轨迹应该是一条**明显的线**

**轨迹特征**：
- **总位移**：较大（从起点到终点的距离较大）
- **单步位移**：每帧之间的位移较大
- **轨迹长度**：在3D空间中，轨迹长度明显

---

## 三、代码逻辑分析

### 3.1 可视化代码

**位置**：`visualize_single_image.py` 第243-260行

```python
for i, (traj, color) in enumerate(zip(traj_world, track_colors)):
    # Create line segments for trajectory
    valid_traj = traj[: gui_track_length.value]  # Use only the first N points
    if len(valid_traj) > 1:  # 至少需要2个点
        segments = []
        seg_colors = []
        for j in range(len(valid_traj) - 1):
            segments.append([valid_traj[j], valid_traj[j + 1]])
            seg_colors.append([color, color])

        if segments:
            track_handle = server.scene.add_line_segments(...)
            track_handles.append(track_handle)
```

**关键点**：
1. **只检查点数**：`len(valid_traj) > 1`（至少需要2个点）
2. **不检查位移**：没有检查轨迹的总位移或单步位移
3. **所有轨迹都会显示**：只要至少有2个点，就会创建line segments

**问题**：
- 即使轨迹位移非常小（接近0），只要至少有2个点，就会显示
- 但在3D可视化中，如果位移太小，轨迹可能看起来像一个点，而不是一条线

---

### 3.2 为什么静止轨迹看起来像点？

**原因**：
1. **位移太小**：如果轨迹总位移 < 0.01米（10毫米），在3D空间中可能不明显
2. **轨迹宽度**：默认轨迹宽度是4.0像素，如果位移太小，可能被轨迹宽度掩盖
3. **点大小**：默认点大小是0.006米，如果轨迹位移接近这个值，可能看起来像一个点

**示例**：
```
静止场景上的keypoint:
  起点: (1.0, 2.0, 3.0)
  终点: (1.001, 2.0, 3.0)  # 位移只有1毫米
  总位移: 0.001米
  
运动物体上的keypoint:
  起点: (1.0, 2.0, 3.0)
  终点: (1.5, 2.3, 3.2)  # 位移明显
  总位移: 0.7米
```

## 七、总结

### 7.1 用户理解的正确性

**✅ 完全正确**：
1. **静止场景上的keypoint**：轨迹长度接近0
2. **运动物体上的keypoint**：轨迹长度明显
3. **可视化中看到的轨迹数量少于400**：因为静止轨迹看起来像点，不明显

### 7.2 关键发现

1. **代码逻辑**：只检查点数（`len(valid_traj) > 1`），不检查位移
2. **可视化效果**：如果轨迹位移太小，在3D空间中可能看起来像一个点
3. **实际数据**：这个case中所有轨迹都有位移，但有些很小

### 7.3 可能的原因

1. **静止场景上的keypoint**：轨迹位移接近0，看起来像点
2. **轨迹重叠**：多个轨迹重叠，看起来像一条
3. **视角问题**：某些轨迹不在视野内或被遮挡
4. **颜色相似**：多个轨迹颜色太接近，难以区分

### 7.4 解决方案

1. **添加位移过滤**：只显示有明显运动的轨迹
2. **添加统计信息**：显示静止轨迹数量
3. **添加可视化选项**：允许用户过滤小位移轨迹
4. **改进颜色映射**：使用更好的颜色映射，区分静止和运动轨迹

---

## 八、关键结论

1. **用户理解正确**：静止场景上的keypoint轨迹长度接近0
2. **可视化效果**：静止轨迹在3D可视化中看起来像点，不明显
3. **代码逻辑**：当前代码不检查位移，所有轨迹都会显示
4. **实际影响**：静止轨迹可能被误认为是keypoint，而不是轨迹
5. **改进建议**：添加基于位移的过滤，只显示有明显运动的轨迹
