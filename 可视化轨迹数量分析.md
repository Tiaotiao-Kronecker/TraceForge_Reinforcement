### 2.2 可能的原因

#### 原因1：轨迹重叠

**问题**：多个轨迹可能重叠在一起，看起来像一条

**示例**：
```
轨迹1: (100, 200) → (101, 201) → (102, 202)
轨迹2: (100, 200) → (101, 201) → (102, 202)  # 完全重叠
轨迹3: (100, 200) → (101, 201) → (102, 202)  # 完全重叠
```

**结果**：3条轨迹看起来像1条

**原因**：
- 查询点均匀分布在整个图像上
- 某些区域的轨迹可能非常接近
- 在3D空间中，轨迹可能重叠或非常接近

---

#### 原因2：轨迹长度限制

**问题**：虽然数据中有128个点，但前30个点可能无效

**代码逻辑**：
```python
valid_traj = traj[: gui_track_length.value]  # 默认30
if len(valid_traj) > 1:  # 需要至少2个点
```

**可能情况**：
- 前30个点中，有些点可能是无效值（`-inf`, `nan`）
- 如果前30个点中只有1个有效点，轨迹不会显示
- 如果前30个点全部无效，轨迹不会显示

---

#### 原因3：轨迹颜色相似

**问题**：多个轨迹颜色太接近，看起来像一条

**代码逻辑**：
```python
track_colors = define_track_colors(traj_world, colormap='turbo')
```

**可能情况**：
- 使用turbo colormap，相邻轨迹颜色可能相似
- 如果轨迹重叠，颜色相似会导致看起来像一条

---

#### 原因4：视角和遮挡

**问题**：某些轨迹可能被遮挡或不在视野内

**可能情况**：
- 轨迹在相机后方（z < 0）
- 轨迹被其他物体遮挡
- 轨迹超出视野范围

---

#### 原因5：轨迹太短或无效

**问题**：虽然数据中有128个点，但前30个点可能无效

**检查逻辑**：
```python
if len(valid_traj) > 1:  # 至少需要2个点
```

**可能情况**：
- 前30个点中，有些点可能是无效值（`-inf`, `nan`）
- 如果前30个点中只有1个有效点，轨迹不会显示
- 如果前30个点全部无效，轨迹不会显示


## 三、Point和Keypoint的定义

### 3.1 Point（点云）

**定义**：从RGB图像和深度图生成的3D点云

**代码位置**：`visualize_single_image.py` 第175-195行

```python
# Create point cloud from RGB image and depth
H, W = depth.shape
points_xyz = unproject_by_depth(
    depth=depth[None, None],  # (1, 1, H, W)
    K=camera_params["K"][None],  # (1, 3, 3)
    c2w=camera_params["c2w"][None],  # (1, 4, 4)
)[0].transpose(1, 2, 0)  # (H, W, 3)

# Downsample point cloud for visualization
downsample_factor = 4
points_xyz_ds = points_xyz[::downsample_factor, ::downsample_factor].reshape(-1, 3)
points_rgb_ds = image[::downsample_factor, ::downsample_factor].reshape(-1, 3)

# Filter out invalid points (depth = 0 or too far)
valid_mask = (points_xyz_ds[:, 2] > 0) & (points_xyz_ds[:, 2] < 10.0)
points_xyz_ds = points_xyz_ds[valid_mask]
points_rgb_ds = points_rgb_ds[valid_mask]

# Add point cloud
point_cloud_handle = server.scene.add_point_cloud(
    name="point_cloud",
    points=points_xyz_ds,
    colors=points_rgb_ds,
    point_size=gui_point_size.value,
    point_shape="rounded",
)
```

**生成过程**：
1. **深度图**：从`depth_path`加载深度数据
2. **RGB图像**：从`image_path`加载RGB图像
3. **反投影**：使用相机内参和外参，将2D像素坐标+深度转换为3D世界坐标
4. **下采样**：每4个像素采样一次（`downsample_factor = 4`）
5. **过滤**：过滤无效点（深度=0或距离>10m）

**特点**：
- **来源**：RGB图像 + 深度图
- **数量**：约 `(H/4) × (W/4)` 个点（下采样后）
- **颜色**：来自RGB图像
- **形状**：rounded（圆角）
- **大小**：由`gui_point_size`控制（默认0.006）

**用途**：
- 显示场景的3D结构
- 提供空间上下文
- 帮助理解轨迹在3D空间中的位置

### 3.2 Keypoint（关键点）

**定义**：轨迹的起始关键点（starting keypoints）

**代码位置**：`visualize_single_image.py` 第262-281行

```python
# Add starting keypoint in 3D (convert from image coordinates to world)
kp_x, kp_y = keypoints[i]  # 从NPZ文件加载的2D关键点坐标
kp_depth = (
    depth[int(kp_y), int(kp_x)] if 0 <= kp_x < W and 0 <= kp_y < H else 1.0
)

# Convert keypoint to world coordinates
kp_world = convert_image_coords_to_world(
    np.array([[[kp_x, kp_y, kp_depth]]]), camera_params
)[0, 0]

keypoint_handle = server.scene.add_point_cloud(
    name=f"keypoint_{i}",
    points=kp_world[None],
    colors=color[None],
    point_size=gui_keypoint_size.value,
    point_shape="circle",
)
keypoint_handle.visible = False  # Initially hidden
```

**生成过程**：
1. **加载关键点**：从NPZ文件的`keypoints`字段加载（形状：`(400, 2)`）
2. **获取深度**：从深度图中获取关键点位置的深度值
3. **坐标转换**：将2D图像坐标+深度转换为3D世界坐标
4. **渲染**：使用轨迹颜色渲染关键点

**特点**：
- **来源**：NPZ文件的`keypoints`字段
- **数量**：400个（对应400个轨迹）
- **颜色**：与对应轨迹颜色相同
- **形状**：circle（圆形）
- **大小**：由`gui_keypoint_size`控制（默认0.005）
- **初始状态**：隐藏（`visible = False`）

**用途**：
- 标记轨迹的起始位置
- 帮助理解轨迹的起点
- 用于调试和分析

## 四、对比总结

### 4.1 Point vs Keypoint

| 特性 | Point（点云） | Keypoint（关键点） |
|------|--------------|-------------------|
| **来源** | RGB图像 + 深度图 | NPZ文件的`keypoints`字段 |
| **数量** | 约 `(H/4) × (W/4)` | 400个（对应400个轨迹） |
| **颜色** | 来自RGB图像 | 与对应轨迹颜色相同 |
| **形状** | rounded（圆角） | circle（圆形） |
| **大小** | `gui_point_size`（默认0.006） | `gui_keypoint_size`（默认0.005） |
| **初始状态** | 显示 | 隐藏 |
| **用途** | 显示场景3D结构 | 标记轨迹起始位置 |

### 4.2 可视化选项

**Point size**：
- **控制对象**：点云（Point）的大小
- **范围**：0.001 到 0.02
- **默认值**：0.006
- **影响**：点云中每个点的大小

**Show keypoints**：
- **控制对象**：关键点（Keypoint）的显示/隐藏
- **默认值**：False（隐藏）
- **影响**：是否显示400个轨迹的起始关键点

## 六、关键结论

### 6.1 轨迹显示数量

**问题**：为什么可视化中看到的轨迹数目没有400个？

**答案**：
1. **轨迹重叠**：多个轨迹重叠在一起，看起来像一条（主要原因）
2. **视角问题**：某些轨迹不在视野内或被遮挡
3. **颜色相似**：多个轨迹颜色太接近，难以区分
4. **轨迹长度**：虽然数据中有128个点，但前30个点可能无效

**实际数据**：
- 所有400个轨迹都有有效点（至少128个）
- 理论上应该都能显示
- 但实际显示可能少于400个，主要原因是轨迹重叠

### 6.2 Point和Keypoint

**Point（点云）**：
- **来源**：RGB图像 + 深度图
- **数量**：约 `(H/4) × (W/4)` 个点
- **用途**：显示场景的3D结构
- **控制**：`gui_point_size`（点大小）

**Keypoint（关键点）**：
- **来源**：NPZ文件的`keypoints`字段
- **数量**：400个（对应400个轨迹）
- **用途**：标记轨迹的起始位置
- **控制**：`gui_keypoint_size`（点大小），`gui_show_keypoints`（显示/隐藏）
