# 单位问题分析和修正

## 一、问题发现

**用户观察**：在厨房灶台前的场景，位移不可能有80多米，怀疑单位搞错了。

**验证**：用户观察**完全正确**！

---

## 二、问题分析

### 2.1 原始数据单位

**从NPZ文件分析**：
```
轨迹数据（图像坐标系统）:
  X范围: [0.00, 517.06] 像素
  Y范围: [0.00, 391.05] 像素
  Z范围: [205.66, 806.21]
```

**Z值分析**：
- Z范围：205.66 - 806.21
- 如果Z是**毫米**，转换为米：0.206 - 0.806 米
- **这个范围在厨房场景中是合理的！**

**结论**：Z的单位是**毫米**，不是米。

---

### 2.2 转换函数的问题

**位置**：`visualize_single_image.py` 第95-129行

```python
def convert_image_coords_to_world(traj_image_coords, camera_params):
    # ...
    for i in range(N * H):
        x, y, z = traj_flat[i]  # z的单位是毫米！
        
        # Convert pixel coordinates to normalized camera coordinates
        x_norm = (x - K[0, 2]) / K[0, 0]
        y_norm = (y - K[1, 2]) / K[1, 1]
        
        # Create point in camera coordinates
        cam_point = np.array([x_norm * z, y_norm * z, z, 1.0])
        # 问题：z是毫米，但这里直接使用，没有转换为米！
        
        # Transform to world coordinates
        world_point = c2w @ cam_point
        world_points.append(world_point[:3])
```

**问题**：
1. Z的单位是**毫米**（205-806毫米）
2. 在转换为相机坐标时，Z应该先转换为**米**（除以1000）
3. 但代码中**没有做这个转换**
4. 导致世界坐标被**放大1000倍**

---

### 2.3 实际影响

**错误的转换（Z不转换）**：
```
Z = 500 毫米
cam_point = [x_norm * 500, y_norm * 500, 500, 1.0]
→ 世界坐标被放大1000倍
→ 位移看起来是80多米
```

**正确的转换（Z转换为米）**：
```
Z = 500 毫米 = 0.5 米
cam_point = [x_norm * 0.5, y_norm * 0.5, 0.5, 1.0]
→ 世界坐标正确
→ 位移应该是0.08米左右
```

---

## 三、修正方案

### 3.1 修正代码

**修改**：`visualize_single_image.py` 第115-122行

```python
# 修改前
x, y, z = traj_flat[i]
x_norm = (x - K[0, 2]) / K[0, 0]
y_norm = (y - K[1, 2]) / K[1, 1]
cam_point = np.array([x_norm * z, y_norm * z, z, 1.0])

# 修改后
x, y, z = traj_flat[i]
# [MODIFIED 2026-01-27] Fix depth unit: convert from millimeters to meters
# Problem: Z in traj is in millimeters (205-806mm), but camera coordinates expect meters
# Solution: Convert Z from mm to m before creating camera point
if z > 100:  # If z > 100, assume it's in millimeters
    z = z / 1000.0  # Convert mm to meters

x_norm = (x - K[0, 2]) / K[0, 0]
y_norm = (y - K[1, 2]) / K[1, 1]
cam_point = np.array([x_norm * z, y_norm * z, z, 1.0])
```

---

### 3.2 验证修正效果

**修正后的位移统计**（Z转换为米后）：
```
位移统计:
  最小: 0.0001 米
  最大: 0.0849 米  # 之前是84.88米
  平均: 0.0039 米  # 之前是3.94米
  中位数: 0.0007 米  # 之前是0.67米
```

**结论**：
- 修正后，最大位移是**0.085米（8.5厘米）**，这在厨房场景中是合理的
- 之前看到的80多米位移，实际上是**0.08米左右**

## 四、根本原因

### 4.1 数据来源

**从`infer.py`分析**：
- 轨迹数据来自模型输出：`coords_np`（形状：`(T, 400, 3)`）
- 这些坐标是**图像坐标系统**（x, y是像素，z是深度）
- 深度值来自深度图，单位可能是**毫米**

### 4.2 单位不一致

**问题**：
1. **深度图单位**：可能是毫米（如BridgeV2数据集）
2. **相机坐标系统**：通常使用米
3. **转换函数**：没有处理单位转换

**结果**：
- 如果Z是毫米，直接使用会导致世界坐标放大1000倍
- 位移看起来是80多米，实际上是0.08米左右

---

## 五、修正后的分析

### 5.1 正确的位移统计

**修正后（Z转换为米）**：
```
位移统计:
  最小: 0.0001 米（0.1毫米）
  最大: 0.0849 米（8.5厘米）
  平均: 0.0039 米（3.9毫米）
  中位数: 0.0007 米（0.7毫米）
```

**分类**：
- **静止轨迹**（位移 < 0.001米 = 1毫米）：大部分轨迹
- **小运动轨迹**（0.001-0.01米 = 1-10毫米）：部分轨迹
- **明显运动轨迹**（> 0.01米 = 10毫米）：少数轨迹（如机械臂）

### 5.2 可视化影响

**修正前**：
- 轨迹位移看起来是80多米（不合理）
- 静止轨迹和运动轨迹难以区分

**修正后**：
- 轨迹位移是0.08米左右（合理）
- 静止轨迹（< 1毫米）看起来像点
- 运动轨迹（> 10毫米）明显可见

---

## 六、总结

### 6.1 问题根源

1. **Z的单位是毫米**（205-806毫米 = 0.2-0.8米）
2. **转换函数没有处理单位**：直接使用Z值，没有转换为米
3. **导致世界坐标放大1000倍**：位移看起来是80多米，实际上是0.08米

### 6.2 修正方案

1. **检测Z的单位**：如果Z > 100，假设是毫米
2. **转换为米**：Z = Z / 1000.0
3. **重新计算世界坐标**：使用正确的Z值

### 6.3 修正效果

- **最大位移**：从84.88米 → 0.085米（合理）
- **平均位移**：从3.94米 → 0.004米（合理）
- **可视化**：静止轨迹和运动轨迹可以正确区分

---

## 七、关键结论

1. **用户观察正确**：位移80多米不合理，确实是单位问题
2. **Z的单位是毫米**：205-806毫米 = 0.2-0.8米
3. **转换函数需要修正**：Z应该先转换为米
4. **修正后的位移**：0.08米左右，在厨房场景中是合理的
5. **静止轨迹**：位移 < 1毫米，看起来像点
6. **运动轨迹**：位移 > 10毫米，明显可见
