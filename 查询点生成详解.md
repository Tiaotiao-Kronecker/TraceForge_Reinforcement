# 查询点（grid_size）生成详解

## 一、纠正之前的误解

**❌ 错误理解**：查询点分布在物体表面

**✅ 正确理解**：查询点**均匀分布在整个图像上**，不区分物体表面还是背景

---

## 二、查询点生成过程

### 2.1 函数定义

**位置**：`infer.py` 第780-811行

```python
def create_uniform_grid_points(height, width, grid_size=20, device="cuda"):
    """Create uniform grid points across the image.
    
    Args:
        height (int): Image height
        width (int): Image width
        grid_size (int): Grid size (20x20)
    
    Returns:
        torch.Tensor: Grid points [1, grid_size*grid_size, 3]
    """
```

### 2.2 生成步骤

#### 步骤1：创建均匀间隔的坐标

```python
# 第793-794行
y_coords = np.linspace(0, height - 1, grid_size)
x_coords = np.linspace(0, width - 1, grid_size)
```

**作用**：
- `np.linspace(0, height-1, 20)`：在Y方向创建20个均匀间隔的点
  - 范围：从0到`height-1`
  - 数量：20个点
  - 间隔：`(height-1) / 19`
- `np.linspace(0, width-1, 20)`：在X方向创建20个均匀间隔的点
  - 范围：从0到`width-1`
  - 数量：20个点
  - 间隔：`(width-1) / 19`

**示例**（假设图像322×518）：
```python
y_coords = [0, 16.9, 33.8, 50.7, ..., 305.1, 322]  # 20个点
x_coords = [0, 27.2, 54.4, 81.6, ..., 490.8, 518]   # 20个点
```

#### 步骤2：创建网格

```python
# 第797行
xx, yy = np.meshgrid(x_coords, y_coords)
```

**作用**：
- 创建20×20的网格
- `xx`：X坐标矩阵（20×20）
- `yy`：Y坐标矩阵（20×20）

**结果**：
```
xx = [[0,   27.2, 54.4, ..., 490.8, 518],
      [0,   27.2, 54.4, ..., 490.8, 518],
      ...
      [0,   27.2, 54.4, ..., 490.8, 518]]  # 20行，每行相同

yy = [[0,   0,   0,   ..., 0,   0],
      [16.9, 16.9, 16.9, ..., 16.9, 16.9],
      ...
      [322, 322, 322, ..., 322, 322]]  # 20行，每行不同
```

#### 步骤3：展平为点列表

```python
# 第800行
grid_points = np.stack([xx.flatten(), yy.flatten()], axis=1)
```

**作用**：
- `xx.flatten()`：将X坐标矩阵展平为400个值
- `yy.flatten()`：将Y坐标矩阵展平为400个值
- `np.stack(..., axis=1)`：组合成400个`(x, y)`坐标对

**结果**：
```
grid_points = [
    [0, 0],      # 左上角
    [27.2, 0],
    [54.4, 0],
    ...
    [490.8, 322], # 右下角附近
    [518, 322]    # 右下角
]  # 共400个点
```

#### 步骤4：添加时间维度

```python
# 第803-804行
time_col = np.zeros((grid_points.shape[0], 1))
grid_points_3d = np.concatenate([time_col, grid_points], axis=1)
```

**作用**：
- 添加时间维度`t=0`（表示第一帧）
- 格式：`[t, x, y]`

**结果**：
```
grid_points_3d = [
    [0, 0, 0],      # [t, x, y]
    [0, 27.2, 0],
    [0, 54.4, 0],
    ...
]  # 共400个点，每个点格式为[t, x, y]
```

## 三、查询点的分布范围

### 3.1 空间范围

**X方向**：
- 最小值：`0`（图像左边缘）
- 最大值：`width - 1`（图像右边缘）
- 覆盖：整个图像宽度

**Y方向**：
- 最小值：`0`（图像上边缘）
- 最大值：`height - 1`（图像下边缘）
- 覆盖：整个图像高度

**结论**：查询点覆盖**整个图像区域**，从左上角`(0, 0)`到右下角`(width-1, height-1)`

### 3.2 分布方式

**均匀网格分布**：
- 20行 × 20列 = 400个点
- 每个网格单元大小：`(width/20) × (height/20)`
- 点与点之间的间隔：`width/19`（X方向）和`height/19`（Y方向）

**示例**（图像322×518）：
- X方向间隔：`518/19 ≈ 27.3`像素
- Y方向间隔：`322/19 ≈ 16.9`像素
- 网格单元大小：约`27.3 × 16.9`像素

### 3.3 可视化示例

```
图像 (322×518):
┌─────────────────────────────────────┐
│ ●  ●  ●  ●  ●  ...  ●  ●  ●  ●  ● │ ← 第1行（y=0）
│                                     │
│ ●  ●  ●  ●  ●  ...  ●  ●  ●  ●  ● │ ← 第2行（y≈16.9）
│                                     │
│ ●  ●  ●  ●  ●  ...  ●  ●  ●  ●  ● │ ← 第3行（y≈33.8）
│                                     │

第1 第2 第3 第4 第5      第16 第17 第18 第19 第20列
```

**说明**：
- `●` 表示查询点
- 均匀分布在整个图像上
- **不区分**物体表面、背景、边缘等

## 四、查询点不区分物体表面

### 4.1 为什么是均匀分布？

**设计原因**：
1. **简单高效**：不需要物体检测或分割
2. **全面覆盖**：确保场景的各个区域都有采样点
3. **通用性**：适用于任何场景，不依赖特定物体

### 4.2 实际效果

**查询点可能落在**：
- ✅ 物体表面（如桌面、物体）
- ✅ 背景区域（如墙壁、地面）
- ✅ 边缘区域（物体边界）
- ✅ 空白区域（无深度信息的地方）

**模型如何处理**：
- 如果查询点落在**有效区域**（有深度值），会被跟踪
- 如果查询点落在**无效区域**（深度为0），可能无法跟踪或轨迹无效

### 4.3 深度值的使用

**在`prepare_query_points`函数中**（第813-832行）：

```python
def prepare_query_points(query_xyt, depths, intrinsics, extrinsics):
    for query_i in query_xyt:
        t = int(query_i[0, 0])
        depth_t = depths[t]
        
        xy = query_i[:, 1:]  # 提取x, y坐标
        ji = np.round(xy).astype(int)
        d = depth_t[ji[..., 1], ji[..., 0]]  # 从深度图查询深度值
        
        # 转换为3D世界坐标
        world_coords = transform_to_world(xy, d, intrinsics, extrinsics)
```

**关键点**：
- 查询点的深度值是从**深度图**中查询的
- 如果该位置没有深度值（深度=0），该点可能无法正确跟踪
- **不预先判断**该点是否在物体表面

## 五、与物体表面的关系

### 5.1 查询点可能落在物体表面

**情况1**：查询点落在物体表面
- 深度值有效（>0）
- 可以成功跟踪
- 轨迹反映物体的运动

**情况2**：查询点落在背景
- 深度值有效（>0）
- 可以成功跟踪
- 轨迹反映背景（通常是静止的）

**情况3**：查询点落在无效区域
- 深度值为0
- 可能无法跟踪
- 轨迹可能被标记为无效

### 5.2 模型如何区分？

**模型不主动区分**，而是：
1. **尝试跟踪所有点**：无论是否在物体表面
2. **使用可见性掩码**：标记哪些点成功跟踪
3. **用户选择**：用户可以根据需要选择感兴趣的轨迹

### 5.3 实际应用

**如果只想跟踪物体表面**：
- 需要**预处理**：使用物体检测或分割
- 或者**后处理**：根据轨迹特征过滤（如运动幅度）
- 或者**手动选择**：从400个点中选择感兴趣的

**当前实现**：
- 跟踪**所有400个点**
- 用户可以根据需要选择使用哪些轨迹

## 六、代码验证

### 6.1 查询点生成代码

```python
# infer.py 第793-800行
y_coords = np.linspace(0, height - 1, grid_size)  # [0, ..., height-1]
x_coords = np.linspace(0, width - 1, grid_size)   # [0, ..., width-1]

xx, yy = np.meshgrid(x_coords, y_coords)
grid_points = np.stack([xx.flatten(), yy.flatten()], axis=1)
```

**验证**：
- `y_coords[0] = 0`：图像上边缘
- `y_coords[-1] = height-1`：图像下边缘
- `x_coords[0] = 0`：图像左边缘
- `x_coords[-1] = width-1`：图像右边缘

**结论**：查询点覆盖整个图像，从边缘到边缘

### 6.2 实际示例

假设图像尺寸为322×518，`grid_size=20`：


## 七、总结

### 7.1 查询点的范围

- **X方向**：从`0`到`width-1`（整个图像宽度）
- **Y方向**：从`0`到`height-1`（整个图像高度）
- **覆盖**：整个图像区域

### 7.2 查询点的分布

- **方式**：均匀网格分布（20×20）
- **数量**：400个点
- **间隔**：`width/19`（X方向）和`height/19`（Y方向）

### 7.3 查询点与物体表面的关系

- **不预先判断**：查询点不区分物体表面还是背景
- **均匀分布**：覆盖整个图像，包括物体、背景、边缘
- **模型处理**：模型尝试跟踪所有点，使用可见性掩码标记成功与否
- **用户选择**：用户可以根据需要选择感兴趣的轨迹

### 7.4 之前的误解纠正

**❌ 错误**：查询点分布在物体表面

**✅ 正确**：
- 查询点**均匀分布在整个图像上**
- 可能落在物体表面、背景、边缘等任何位置
- 模型不区分，尝试跟踪所有点
- 最终结果中，只有成功跟踪的点才有有效轨迹
