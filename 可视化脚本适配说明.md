# 可视化脚本适配说明

## 一、问题分析

### 1.1 适配需求

**问题**：将`grid_size`从20改为80后，轨迹数量从400增加到6400（16倍），导致可视化性能问题。

**影响**：
1. **渲染性能**：6400个轨迹线渲染非常慢
2. **内存占用**：内存占用增加16倍
3. **交互性能**：界面卡顿，难以操作
4. **可视化效果**：轨迹重叠严重，难以区分

---

## 二、适配方案

### 2.1 添加的功能

#### 1. GUI滑块控制

**新增GUI控件**：
- **Max trajectories**：控制显示的最大轨迹数（默认1000）
- **Max keypoints**：控制显示的最大keypoint数（默认1000）

**位置**：`visualize_single_image.py` 第234-240行

```python
gui_max_trajectories = server.gui.add_slider(
    "Max trajectories", min=100, max=len(traj_world), step=100, initial_value=default_max_traj
)
gui_max_keypoints = server.gui.add_slider(
    "Max keypoints", min=100, max=len(keypoints), step=100, initial_value=default_max_kp
)
```

---

#### 2. 轨迹下采样

**实现方式**：均匀采样轨迹

**位置**：`visualize_single_image.py` 第260-284行

```python
# Downsample trajectories and keypoints
if len(traj_world) > max_traj_display:
    # Uniformly sample trajectories
    indices = np.linspace(0, len(traj_world) - 1, max_traj_display, dtype=int)
    traj_world_display = traj_world[indices]
    track_colors_display = track_colors[indices]
    keypoints_display = keypoints[indices]
    logger.info(f"Downsampled trajectories: {len(traj_world)} → {max_traj_display} for visualization")
```

**特点**：
- 均匀采样，保持空间分布
- 自动记录下采样信息
- 支持动态调整（通过GUI滑块）

---

#### 3. Keypoint下采样

**实现方式**：独立下采样keypoint

**位置**：`visualize_single_image.py` 第280-284行

```python
# Further downsample keypoints if needed
if len(keypoints_display) > max_kp_display:
    kp_indices = np.linspace(0, len(keypoints_display) - 1, max_kp_display, dtype=int)
    keypoints_display = keypoints_display[kp_indices]
    logger.info(f"Downsampled keypoints: {len(keypoints)} → {max_kp_display} for visualization")
```

**特点**：
- 独立控制keypoint数量
- 可以显示比轨迹更少的keypoint

---

## 三、使用方式

### 3.1 默认使用

**命令**：
```bash
python visualize_single_image.py \
    --npz_path <path> \
    --image_path <path> \
    --depth_path <path>
```

**效果**：
- 自动下采样到1000个轨迹（如果数据有6400个）
- 自动下采样到1000个keypoint（如果数据有6400个）

**命令**：
```bash

    --npz_path <path> \
    --image_path <path> \
    --depth_path <path> \
    --max_trajectories 2000 \
    --max_keypoints 2000
```

**效果**：
- 显示2000个轨迹
- 显示2000个keypoint
- 性能根据数量调整

---

### 3.3 GUI动态调整

**操作**：
1. 打开可视化界面
2. 在GUI中找到"Max trajectories"滑块
3. 调整滑块值（100-6400）
4. 轨迹数量自动更新

**效果**：
- 实时调整显示数量
- 无需重启可视化
- 方便探索不同密度

## 七、注意事项

### 7.1 下采样方式

**当前实现**：均匀采样（`np.linspace`）

**特点**：
- 保持空间分布均匀
- 简单高效
- 可能丢失某些重要轨迹

**替代方案**（如果需要）：
- 基于轨迹长度的采样（优先显示长轨迹）
- 基于位移的采样（优先显示运动轨迹）
- 随机采样

---

### 7.2 默认值设置

**当前默认**：1000个轨迹/keypoint

**原因**：
- 平衡性能和可视化效果
- 对于6400个轨迹，1000个已经足够展示主要特征
- 性能可接受

**调整建议**：
- 如果性能仍然不足：减少到500或更少
- 如果需要查看所有轨迹：增加到6400

---

### 7.3 兼容性

**向后兼容**：
- 对于400个轨迹的数据，仍然正常工作
- 默认值自动适配数据大小
- 不影响现有使用方式
