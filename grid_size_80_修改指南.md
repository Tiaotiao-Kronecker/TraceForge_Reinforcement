- `grid_size = 20`（20×20 = 400个查询点）

### 1.3 支持点调整（support_grid_size）

#### 位置6：`infer.py` 第547行

```python

```

**修改选项**：

**选项1：保持比例（推荐，如果计算资源充足）**
```python
support_grid_size=64,  # 16 * (80/20) = 64
```
- 支持点：64 × 64 = 4096 点
- 优点：保持查询点和支持点的比例，跟踪质量最好
- 缺点：计算量增加（16倍）

**选项2：适度增加（推荐，如果计算资源有限）**
```python

```
- 支持点：32 × 32 = 1024 点
- 优点：计算量适中，跟踪质量仍然较好
- 缺点：比例略低于原始设置

**选项3：保持固定值（不推荐）**
```python

```
- 支持点：16 × 16 = 256 点
- 优点：计算量最小
- 缺点：查询点增加了4倍，但支持点不变，可能导致跟踪质量下降

**推荐**：使用选项1（如果计算资源充足）或选项2（如果计算资源有限）
